# MSIMPL - Microservice Implementation Standard## PurposeDefine standard practices for observability, traceability, and Kafka-based event handling.## Traceability- traceId generated at entry point (Controller/Gateway)- propagated via Kafka headers- stored in MDC- cleared after request## Metrics- Standard metric names across all services- Tags: service, topic- No high-cardinality tags## Logging- Logback includes service and traceId- Structured logs only## Producers- Extend BaseKafkaProducer- Use trace-aware record building- Emit success/failure metrics## Consumers- Extract traceId from headers- Emit metrics and latency- Clear MDC in finally block## Controllers- Manage trace lifecycle only- No business logic## DLQ- Retry → DLQ- Replay supported- Query supported
